  • Unlike other components, the masonry grid utilizes the embed tag.
  • Inside of the {% verbatim %}{% block list %} {% endblock %}{% endverbatim %} tag data is being passed to the masonry.html template partial. There is no set amount of entries you can have inside of a grid. It is suggested however you have at least three rows for the sorting to be effective.
  • Inside of .masonry__item the spotlight component or custom content should be included.
  • Set shuffle to true to enable shuffling. Otherwise the masonry grid will be loaded in order.
{% markdown %} ```twig.line-numbers {% verbatim %} {# // Default template structure #} {% embed "_components/masonry/content.html" with { "shuffle": false, "filters": masonryFilters } %} {% block list %}
...
...
...
...
{% endblock %} {% endembed %} {# // An example using spotlight #} {% embed "_components/masonry/masonry.html" with { "shuffle": false, "filters": masonryFilters } %} {% block list %}
{% include "_components/spotlight.html" with { ... } %}
{% include "_components/spotlight.html" with { ... } %}
{% endblock %} {% endembed %} {% endverbatim %} ``` {% endmarkdown %}
  • Filtering is optional.
  • To add filtering, you first must define a variable of masonryFilters with data.
  • The .filter-- value can be anything. Ideally you want to use a single letter word that is relevenat to your category. As an example: .filter--media, .filter--culture, .filter--social, etc
  • These values will be used to populate the filter select dropdown.
{% markdown %} ```twig.line-numbers {% verbatim %} {# // Filtering data #} {% set masonryFilters = [ { "value": ".filter--categorySlug1", "label": "Category Name 1" }, { "value": ".filter--categorySlug2", "label": "Category Name 2" }, { "value": ".filter--categorySlug3", "label": "Category Name 3" }, { "value": ".filter--categorySlug4", "label": "Category Name 4" } ] %} {# // Once the values are defined, the .filter-- should be used on the .masonry__item container #}
...
...
{% endverbatim %} ``` {% endmarkdown %}
