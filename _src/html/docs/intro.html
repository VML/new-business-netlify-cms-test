Intro

  • Primary introductory banner that should define the purpose of the stage site.
  • Must contain h1 element
  • Select from pre-defined banner images or use custom image related to potential new client.
  • The banner image itself should not contain text if at all possible. Reason being is that the banner image fills up a specified width/heigth in the banner. Any text at the edges of the banner could get cut off.
  • ParticleJS uses javascript to add customizable interactive or non-interactive particles to the background. Set the variable to true {% verbatim %} {% set enableParticle = true %} {% endverbatim %} on the page with the ParticleJs background to enable the javascript in the base layout. See the ParticleJS demo for settings avaialble.
Key Value Default
title The primary title used to describe the stage site. --
titleAbove Smaller title that appears above the larger title --
titleBelow Smaller title that appears below the larger title --
bgFileName The name of your background image. Please be sure to place image under /_src/images/ --
classes Apply utility classes to format banner. particle__fg (particles in the foreground) or particle__bg (particles in the background behind text and components) is required for the ParticleJS background. vg-4 color-white
buttons Button that appears below the title. Can be used to link out to documents. Each button entry supports: title, url, icon, whether the link should open in a new window, and class for the button color. --
id Id is required with the ParticleJS background but can be used for anything requiring an id. --

ParticleJs Background

  • The /static/assets/js/particles-app.js file contains the configuration parameters.
  • Each background requires a unique id with its own configuration parameters.
  • Use the class particle__fg to put the particle canvas infront of the content so it is interactive or particle__bg to put it behind the content.
